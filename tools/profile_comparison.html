Both curves are normalised so their mean speed over one full cycle equals 1 — since both profiles run the same total sweep distance in the same Sweep time, this makes them directly comparable. Speed is signed: positive while sweeping back→forward, negative on the return leg. Sine follows a cosine taper (speed never drops below 20% of centre speed); Reciprocal follows an exponential-decay taper (down to 10%) — so Reciprocal slows more aggressively near the wafer edge and runs faster through the centre to compensate.

S-curve endpoint ramp (Accel/Jerk): at every direction reversal the arm decelerates to a complete stop and then re-accelerates using a 7-phase S-curve profile — so velocity passes smoothly through zero at each edge. Accel (°/s²) caps the peak acceleration/deceleration. Jerk (°/s³) limits how fast acceleration can change: lower jerk gives a gentler onset/exit, higher jerk gives snappier transitions. The peak cruise speed vPeak is found by binary search so the total cycle time stays exactly equal to Sweep time — a wider ramp requires a higher vPeak through the centre to compensate. The ramp distance shown in the stats is how many degrees the arm needs to accelerate from rest to vPeak under the S-curve constraints.

Acceleration (top chart) shows d(speed)/dt. Because velocity now passes smoothly through zero at each reversal, the acceleration chart shows smooth S-curve ramps instead of sharp spikes.
